# OCR検索 / ファイル名検索

| 要件 | 値 |
|----|----|
| ログインの有無 | True |
| エンドポイント | /v1/search |
| メソッド | GET |

## 概要

ファイル名および画像内テキスト（OCR）を対象に統合検索する。
ユーザーは検索タイプを意識せず、`q` パラメーターにキーワードを指定するだけでよい。

### 最大検索可能文字数

255文字まで

### 対象

- ファイル名（部分一致、大文字小文字無視）
- OCR 抽出テキスト（部分一致、大文字小文字無視）

## クエリパラメーター

| キー | 値の種類 | 必須 | デフォルト | 説明 |
|------|---------|------|-----------|------|
| q | String | True | - | 検索キーワード（最大255文字）|
| page | Integer | False | 1 | ページ番号 |
| limit | Integer | False | 50 | 件数（最大: 100）|

## レスポンス

| キー | 値の種類 | 説明 |
|------|---------|------|
| files | Array | ファイルオブジェクトの配列 |
| total | Integer | 全件数 |
| page | Integer | 現在ページ番号 |
| limit | Integer | 1ページ件数 |

## 実装要件

- PostgreSQL の `filename ILIKE` および `ocr_text ILIKE` で OR 検索
- OCR テキストは画像アップロード後に非同期（Apalis ジョブキュー）で生成
- 必要コンポーネント: PostgreSQL、ndlocr-lite（詳細: [OCRエンジン](../../spec/infra/ocr.md)）、Apalis + Redis
